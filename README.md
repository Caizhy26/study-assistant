# 学习助手项目（已改为后端代理版）

这个版本已经修掉了两个核心问题：

1. 不再由前端浏览器直接携带 API Key 请求第三方模型。
2. 前端统一走 `/api/chat` 和 `/api/vision`，部署后只要后端可用，就能真正连上 API。

## 为什么你之前“生成网址后连不上 API”

因为之前是纯前端直连第三方接口，并把 Key 存在浏览器本地。这种做法在正式部署后很容易出现：

- Key 暴露在客户端，不安全
- 静态网站没有后端，无法安全保管 Key
- 文本接口和视觉接口还是两套配置，导致“文字能用、图片不在线”

现在已经改成：

- 前端：只请求你自己的 `/api/*`
- 后端：读取服务端环境变量里的 Key，再去请求真实模型接口

## 本地启动

先准备环境变量：

1. 复制 `.env.example` 为 `.env`
2. 至少填写 `DEEPSEEK_API_KEY`
3. 如果你需要图片识别，再填写 `VISION_API_KEY`

然后分别开两个终端：

### 终端 1：启动后端代理

```bash
npm run dev:backend
```

默认端口：`http://localhost:8787`

### 终端 2：启动前端

```bash
npm run dev:frontend
```

默认端口：`http://localhost:52475`

Vite 已经配置了 `/api` 代理，所以前端开发时会自动转发到后端。

## 部署说明

### 方案 A：前后端同域部署

最省心。前端继续请求 `/api/chat` 和 `/api/vision`，由你的服务器或反向代理把 `/api/*` 转发给 `server/index.mjs`。

### 方案 B：前后端分开部署

- 前端单独部署
- 后端单独部署 `server/index.mjs`
- 前端环境变量里设置：

```bash
VITE_API_BASE_URL=https://你的后端域名
```

这样生成的网址就会真正打到你的后端，而不是在浏览器里裸连第三方模型。
